# Чек‑лист оператора Reality3D (магазин, склад, касса, логи)

Этот документ описывает ежедневные операции для полноценной работы магазина и производственных процессов.

## 0) Быстрая проверка перед началом смены (5 минут)

1. Откройте админку: `/admin`.
2. Проверьте кассу: `/admin/finance`
   - Балансы трёх касс: Онлайн‑касса / Касса офиса / Банковский счёт.
3. Проверьте склад: `/admin/warehouse`
   - Низкие остатки: `/admin/warehouse/low-stock`
4. Проверьте логи: `/admin/logs`
   - Нет ли подозрительной активности / ошибок операций.

## 1) Товары: создание/редактирование

### Создать товар
1. `/admin/shop/products/new`
2. Заполните:
   - Название, SKU (если используете), цена, остаток (если стартовый).
   - `isActive = true`, чтобы товар был виден в магазине.
3. Сохраните.
4. Проверьте карточку товара в магазине: `/shop`.

### Обновить товар
1. `/admin/shop/products`
2. Откройте товар и внесите правки.

## 2) Склад: приход товара (оприходование)

Вариант A (рекомендуется): **Приходы** (документ прихода).

1. Создайте приход: `/admin/warehouse/receipts`
2. Откройте приход и добавьте позиции (SKU/наименование/кол‑во/себестоимость).
3. Нажмите **Провести**.
4. Проверьте, что:
   - Остаток по товару вырос (каталог склада / карточка товара).
   - В логах появилась операция **Приход**: `/admin/logs` (тип: Склад).

Вариант B: **Операции склада** (простое движение).

1. `/admin/warehouse/operations`
2. Выберите действие **Приход**, товар, количество.
3. Проведите операцию.

## 3) Склад: списание/перемещение/инвентаризация

### Списание
1. `/admin/warehouse/operations`
2. Выберите действие **Списание** + причину (брак/внутреннее/продажа/и т.д.).
3. Проведите.
4. Проверьте, что свободный остаток уменьшился, а в логах появилась операция **Списание**.

### Инвентаризация
1. `/admin/warehouse/inventory`
2. Создайте инвентаризацию, внесите фактические количества.
3. Нажмите **Провести**.
4. Проверьте:
   - Остатки приведены в соответствие.
   - Логи: действие **Инвентаризация**.

## 4) Заказ магазина: обработка, оплата, отгрузка

### A) Новый заказ (создан на checkout)
1. `/admin/shop/orders`
2. Откройте заказ.
3. Проверьте:
   - Состав заказа
   - Данные покупателя и доставки
   - Статусы: заказ / оплата / доставка

### B) Ручная оплата (Т‑Банк перевод по ссылке)
Используется сценарий, когда клиент платит по ссылке/QR, а оператор подтверждает факт поступления.

1. Откройте заказ: `/admin/shop/orders/[id]`
2. Нажмите **Подтвердить оплату**
3. Система автоматически:
   - Ставит оплату и заказ в статус **Оплачен**
   - Создаёт запись платежа
   - Делает списание со склада по позициям заказа
   - Создаёт приход в кассу (как `Оплата заказа`)
4. Проверьте:
   - `/admin/shop/payments` — есть платеж по заказу
   - `/admin/finance` — есть операция по кассе
   - `/admin/logs` — есть записи списания/движений

### C) Отгрузка
1. В заказе заполните:
   - Статус доставки (например **Отправлен**)
   - Перевозчик и трек‑номер (если есть)
2. Сохраните.
3. Проверьте, что клиент видит обновления на странице заказа.

## 5) Касса: ручные операции и сверка

### Ручная операция (приход/расход)
1. `/admin/finance`
2. В блоке **Добавить операцию**:
   - Касса
   - Тип (Приход/Расход/Перевод/Коррекция)
   - Код (например `manual_cash`)
   - Сумма и комментарий
3. Нажмите **Создать**

### Сверка кассы
1. `/admin/finance/reconciliations`
2. Выберите кассу и день.
3. Укажите факт (если есть), добавьте комментарий.
4. Проверьте, что статус сверки корректный.

## 6) Заказ из калькулятора (3D‑печать)
1. Клиент создаёт заказ в ЛК: `/lk/orders`
2. Оператор обрабатывает заказ: `/admin/orders`
3. Статусы и комментарии ведутся внутри заказа.

## 7) “Оставить заявку”
Форма “Оставить заявку” отправляет уведомления в ТГ и Max и **не должна** создавать сущности в базе данных.

## 8) Полезные страницы
- Магазин: `/shop`
- Checkout: `/checkout`
- Заказы магазина (админ): `/admin/shop/orders`
- Платежи (админ): `/admin/shop/payments`
- Склад: `/admin/warehouse`
- Приходы: `/admin/warehouse/receipts`
- Операции склада: `/admin/warehouse/operations`
- Касса: `/admin/finance`
- Логи: `/admin/logs`
