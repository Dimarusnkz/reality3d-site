## 2) Найденные проблемы

### 2.1 Риск: секрет в репозитории
- Был обнаружен файл `debug-telegram.js` с хардкодом Telegram Bot Token.
- Исправление: файл удалён.
- Рекомендация: перевыпустить Telegram Bot Token (так как он мог утечь из репозитория/логов).

### 2.2 Жёсткая привязка к складу `warehouseId = 1`
- В нескольких местах логики была привязка к `warehouseId = 1`, что ломает работу при изменении/миграции складов.
- Исправление: добавлен выбор “основного склада” по коду `main` (fallback на первый существующий, иначе создание).

### 2.3 Seed не создавал базовые сущности
- Без базового склада/касс/permissions запуск админских процессов нестабилен.
- Исправление: `prisma/seed.js` расширен — создаёт склад `main`, кассы, permissions и базовые role permissions.

## 3) План исправлений (если появятся новые проблемы)

1. Любая ошибка бизнес‑процесса → воспроизведение через smoke‑скрипт или unit‑тест.
2. Исправление в server actions / API routes.
3. Деплой и повторный smoke на сервере.
4. Обновление чек‑листа оператора и тест‑плана при изменении процесса.
